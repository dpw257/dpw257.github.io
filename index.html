Data analysis and visualisation


The challenge of data analysis is to think beyond the obvious. By knowing how to ask the right questions, I'm able to uncover meaningful relationships in my data, which is why I strive to gain a solid understanding of the scope of the project and business context. I select my methods and tools according to the task, for example, I'd use time series analysis for forecasting market trends, while I'd conduct multiple linear regression to identify the causes of inefficiency on a production line. By employing statistical techniques such as calculating confidence intervals, I'm able to quantify the reliability of my conclusions. To visualise my results, I use tools like Power BI to create reports and dashboards and Python libraries to generate effective charts. I prioritise presenting data in a way that reflects the expertise and time constraints of the audience, highlighting the key findings that reveal the story behind data.


Data engineering and quality


There is no one-size-fits-all solution to data problems. Designing an effective approach is highly task-specific and depends on users' expectations regarding processing, whether they need to monitor data flows live or intend to conduct in-depth analyses on large datasets. I can integrate different AWS services, from Amazon MSK and Redshift, into a pipeline to handle real-time data and ensure robust storage, with high performance and the possibility to scale as needed. I also have experience automating repetitive tasks using DAGs on Apache Airflow. Since data quality is a critical aspect of any data pipeline, I implement validation checks early to detect anomalies, clean data before processing and use monitoring tools to identify and resolve issues swiftly.
