Data Engineering

PostgreSQL Database for Sales Records


A multinational company stored its customer, sales and revenue data in different locations and formats. The data was extracted, cleaned and converted into a centralised database to enable easy access and allow the company to make better, data-driven decisions.

1. Project objectives

  ●   Extract and clean data from AWS RDS database, RESTful API, PDF, JSON and CSV files.
  ●   Load the data into a PostgreSQL database with dimensions tables.
  ●   Run SQL queries to extract business insights on sales and revenue.


2. Methodology

2.1. Dataset


Six tables with data on customers, stores, products, orders, dates, and credit cards were accessed from an RDS database, an API and an S3 bucket on AWS Cloud, as well as via direct links to files.


2.2. Extracting


A class of methods was defined to access the AWS server and extract each file type as a Pandas dataframe. Access credentials were read securely from a .yaml file. The S3 bucket was accessed using boto3, and the PDF was read with tabula in Python.



Method for connecting to AWS RDS database


2.3. Cleaning


Methods were also defined to clean each file, which included dropping duplicate and empty rows, dropping or merging columns, standardising units, and using regex to ensure the correct format of numbers, dates and email addresses.



Checking date format with regex


2.4. Uploading and querying


A database was compiled from the dataframes by defining primary and foreign keys, and the database was uploaded to PostgreSQL locally. Then SQL queries were run using SQLAlchemy in Python to gain insights on sales and revenue per store and per country.



Defining primary and foreign keys for database



Example SQL query


3. Results and insights

The database achieved good latency of 0.01, showcasing its efficiency. The system is easy to scale since since the functions designed for data extraction and cleaning can easily accommodate new data, which can then be added to the database for further analysis.
